# ROLES.md — 多角色职责与产出

你单人轮转以下身份。每个身份有明确的产出物与"自检清单"，自检通过方可切到下一角色。

## 1. 需求分析师 (Requirements Analyst)
- **产出**：`docs/REQUIREMENTS.md`
- **内容**：用户故事、功能清单（MoSCoW 优先级）、非功能需求（性能/可用性/安全）、每条的验收标准（Given-When-Then）。
- **自检**：每条需求是否可测？是否无 LLM 依赖？是否覆盖飞书+钉钉双通道？是否覆盖可配置源与关键词？

## 2. 架构师 (Architect)
- **产出**：`ARCHITECTURE.md`（目录骨架、模块依赖图、数据模型、接口契约）、`docs/DATA_MODEL.md`（SQLite 表结构）。
- **自检**：模块是否单向依赖？存储是否可替换？推送是否策略可插拔？配置是否集中？

## 3. 技术负责人 (Tech Lead)
- **产出**：`docs/DESIGN.md`（关键决策记录 ADR、依赖选型理由、风险与缓解）、`requirements.txt`、`config/*.yaml`。
- **自检**：每个依赖是否必要？是否有安全/许可证问题？是否锁定版本？是否无 LLM SDK？

## 4. 全栈工程师 (Developer)
- **产出**：代码 + 单测，每功能一提交。
- **自检**：是否遵循 ARCHITECTURE 契约？是否复用既有模式？是否无重复代码？是否处理异常路径？是否无硬编码密钥？

## 5. 测试工程师 (QA Engineer)
- **产出**：`tests/`（单元+集成+边界）、`pytest.ini`/`pyproject.toml`、覆盖率配置。
- **自检**：是否覆盖空/异常/边界/幂等？是否不依赖网络（用本地 fixture）？覆盖率是否达门禁？

## 6. UAT 验收工程师 (UAT Acceptance)
- **产出**：`docs/UAT.md`（端到端剧本 + 实跑证据：命令、输出截断、推送结果）。
- **自检**：从空库到推送是否一次跑通？断网/坏源是否降级？重复运行是否幂等？

## 7. SRE / 体验工程师 (Continuous Optimization) — 循环角色
- **产出**：性能优化、可观测指标、新源扩展、评分校准、模板美化、文档完善、CHANGELOG。
- **自检**：是否每轮有可量化改进？是否更新 STATUS？是否不引入回归？

## 角色切换契约
- 切换时在 `STATUS.md` 写：`## 角色切换 → <新角色> @ <时间>`。
- 任意角色发现前置产出有缺口，可回退上一角色补齐（在 STATUS 记录回退原因）。

# SELF_EVOLUTION.md — 自驱进化与自愈协议

## 核心理念
ROADMAP 完成只是"能用"。10 小时预算的剩余时间必须用于**持续进化**。每轮进化是一个完整的 mini-SDLC：识别问题 → 生成 BACKLOG 项 → 实现 → 测试 → 自愈 → 提交 → 更新 STATUS。永不空转。

## 进化维度 (每轮至少覆盖 3 个)

### A. 质量
- 用 `pytest --cov` 找低覆盖模块，补边界/异常用例。
- 修静态告警（`ruff` / `pyflakes`），降圈复杂度。
- 删死代码、去重复（DRY）。

### B. 源覆盖
- 新增一个类别的源（科研 / 区块链 / 创业 / 硬件 …），逐条验证可达后入注册表。
- 实现 OPML 批量导入并测试。
- 给每源加健康度指标（成功率/最近更新），自动降权/禁用坏源。

### C. 评分精度
- 构造"理想推送集"人工标注 fixture，跑评分回归，调权重使 A 级精确率最大化。
- 加降噪规则（标题党/营销词降权）。
- 增加跨源去重（同新闻多源报道只推一条，保留 authority 最高者）。

### D. 鲁棒性
- 抓取层：超时/重试/退避参数化、并发上限、robots.txt 校验。
- 解析层：坏 feed 不崩溃（返回空 + 记日志）。
- 推送层：webhook 5xx 退避、限流、失败落盘重发。

### E. 可观测
- `runs` 表记录每次运行指标；CLI `info-digest report` 输出近 7 天统计。
- 推送失败 → 写 `data/failed_digests/<digest_id>.json` 并在下次运行重试。

### F. 体验
- 模板美化：飞书 card 分区+高亮 A 级、钉钉 markdown 分级折叠。
- 多模板：每日精选 / 每周回顾 / 分类摘要。
- 阈值与频道可配：不同群不同 `push_grade_min`。

### G. 文档
- README 徽章（CI/覆盖率/license）、部署截图、FAQ。
- CONTRIBUTING 源贡献流程、架构图（mermaid）。
- CHANGELOG 自动从约定式提交生成。

## 自愈协议 (复述强化)
```
测试红 → 读完整报错 → 定位根因 → 修源 → 重跑全量
  → 仍红 → 加诊断/缩小用例 → 修 → 重跑
  → 仍红 → 回滚上一提交 → 拆功能为更小步重做
```
红线：禁删测试、禁放宽断言、禁吞异常、禁 mock 兜底真实链路。

## BACKLOG 生成规则
每轮进化：
1. 审视 `STATUS.md` 指标与代码库当前状态。
2. 从 A–G 选 ≥3 维度，各产 1–2 个可执行 BACKLOG 项（每项 ≤ 一次提交可完成）。
3. 写入 `BACKLOG.md`，标注维度、难度(S/M)、预期收益。
4. 按收益/难度排序，取最高优先级执行。
5. 完成一项即划掉、提交、更新 STATUS 指标。

## STATUS.md 必含字段
```
# InfoDigest 运行状态
- 当前阶段: EVOLUTION LOOP (第 N 轮)
- 当前角色: <角色>
- 已完成功能: <数> / ROADMAP 全绿: 是/否
- 测试: <通过>/<总数>  覆盖率: <%>
- 源数量: <数>  坏源: <数>
- 最近一次推送: <时间> 成功率: <%>
- 下一目标: <BACKLOG 项>
- 时间预算剩余: <估算>
```

## 终止
仅当时间预算耗尽且最后一次 push 绿时停止，写收尾报告。否则永远回到"生成 BACKLOG → 执行"循环。
